No-trade is a decision, not a blacklist.

Capture broadly, model selectively, execute narrowly. This blueprint separates companies worth observing from events the generic earnings engine can interpret and securities that are actually tradable at that moment.

Keep the evidence, withhold the setup.

A company can be useful to the observation run and still be ineligible for the generic model or for a hypothetical trade. Every exclusion is effective-dated, reason-coded, and replayable.

The forward run must include no-trade controls. Filtering them out during capture would hide exactly the failures we need to measure.

Three nested universes

The labels answer different questions. Conflating them would either discard research evidence or allow an unsupported model to create an intent.

1

Capture universe

Retain releases, filings, source timing, price discovery and negative controls. This is intentionally broad.

2

Model-eligible universe

Only issuers with the correct metric packet, expectation basis and validated model family advance.

3

Event-time trade universe

The exact security must also pass liquidity, quote, halt, contamination, latency and post-cost gates.

Six ordered gates

The first failure is the primary reason; later failures remain contributing reasons. An unknown input fails closed.

Gate 1

Security & venue

Common equity, supported listing, deterministic identity.

Gate 2

Executable market

Price, dollar capacity, fresh quote, spread, depth, halt, borrow.

Gate 3

Evidence integrity

Prospective expectations, period, market snapshot, revision identity.

Gate 4

Model-family fit

Correct sector economics and external-driver controls.

Gate 5

Clean event

No independent catalyst or unstable price discovery.

Gate 6

Edge after costs

Walk-forward evidence, executable fill and positive net expected value.
